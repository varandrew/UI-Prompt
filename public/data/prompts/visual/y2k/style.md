# Style Prompt

## 中文版本 (zh-CN)

Y2K 风格指南：
- 使用镀铬/玻璃渐变、高饱和粉/蓝绿/紫色，背景可配网格、星光或扫描线。
- 字体选窄体无衬线或像素/科技感字体，按钮与窗口带高光和微发光描边。
- 结构包含顶部导航、Hero、功能/价格卡、复古窗口或播放器组件，留出明显圆角。
- hover/focus 添加霓虹外发光或轻微闪烁/扫描线，保持文本对比与无障碍状态。

## English Version (en-US)

Y2K style guide:
- Use chrome/glass gradients with vibrant pink/teal/purple; backgrounds may include grids, stars, or scan lines.
- Choose condensed sans or pixel/tech fonts; buttons/windows carry highlights and mild neon strokes.
- Layout includes top nav, hero, feature/pricing cards, and retro window or player elements with clear radii.
- Hover/focus adds neon outer glow or subtle flicker/scan-line effects while keeping readable contrast and accessibility.

## English Version (en-US)

**Style Guide & Implementation**

**Core Principles**: This style emphasizes visual distinctiveness, technical excellence, and user-centered design. Follow established patterns while allowing creative expression within defined constraints.

**Visual Language**: Establish clear visual vocabulary through consistent use of colors, shapes, spacing, and typography. Create recognizable patterns that users can learn and predict.

**Technical Implementation**: Use modern CSS techniques including Grid, Flexbox, Custom Properties, and advanced selectors. Optimize for performance using GPU-accelerated properties and efficient rendering strategies.

**Color Application**: Apply colors purposefully with semantic meaning. Ensure sufficient contrast for accessibility (WCAG AA minimum). Use color to guide attention and communicate state changes.

**Typography System**: Implement hierarchical type scale with clear differentiation between levels. Use appropriate font families, weights, and spacing for optimal readability and visual interest.

**Spacing & Rhythm**: Maintain consistent spacing rhythm using modular scale. Apply generous whitespace to improve scannability and reduce cognitive load. Create visual breathing room around important elements.

**Interaction Design**: Provide clear, immediate feedback for all user interactions. Use appropriate hover, focus, and active states. Ensure interactions are intuitive and discoverable.

**Responsive Adaptation**: Design layouts that adapt gracefully to different screen sizes. Prioritize content and functionality on smaller screens. Enhance experience on larger displays.

**Animation & Motion**: Use motion purposefully to guide attention, provide feedback, and create delight. Keep animations subtle and performant. Respect user preferences for reduced motion.

**Accessibility Standards**: Ensure all users can access and interact with content regardless of ability. Provide keyboard navigation, screen reader support, and sufficient contrast. Follow WCAG 2.1 guidelines.

**Performance Targets**: Optimize for fast initial load (<3s), smooth interactions (60fps), and minimal resource consumption. Test on mid-range devices to ensure broad compatibility.

**Browser Support**: Ensure compatibility with modern browsers while providing graceful degradation for older versions. Use feature detection and progressive enhancement strategies.
